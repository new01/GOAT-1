  <name>Task 2: Create Convex schema and builder persistence functions</name>
  <files>convex/schema.ts, convex/builders.ts</files>
  <action>
    **Step 1: Initialize Convex**
    Run: `npx convex dev` -- this will prompt for login/signup on first run, create a development deployment, and generate `.env.local` with `NEXT_PUBLIC_CONVEX_URL`. Let the interactive setup complete, then Ctrl+C (we just need the schema pushed, not the long-running dev server).

    If `npx convex dev` hangs waiting for changes, use `npx convex deploy` or `npx convex dev --once` instead to just push the schema.

    **Step 2: Create Convex schema**
    Create `convex/schema.ts`:
    ```typescript
    import { defineSchema, defineTable } from "convex/server";
    import { v } from "convex/values";

    export default defineSchema({
      builders: defineTable({
        walletAddress: v.string(),
        completedSteps: v.array(v.number()),
        currentStep: v.number(),
        startedAt: v.optional(v.number()),
        completedAt: v.optional(v.number()),
      }).index("by_wallet", ["walletAddress"]),
    });
    ```

    Fields:
    - `walletAddress`: string, indexed for fast lookup (lowercased for consistency)
    - `completedSteps`: array of step numbers (0-4) that are complete
    - `currentStep`: current active step number (0-4)
    - `startedAt`: optional timestamp (epoch ms) when wallet first connected (set in Phase 2)
    - `completedAt`: optional timestamp when all steps done

    **Step 3: Create builder query and mutation functions**
    Create `convex/builders.ts` with three functions:

    1. `getByWallet` (query): Takes `walletAddress` string, uses `withIndex("by_wallet")` to look up builder, returns the builder doc or null. Lowercases the wallet address before querying for consistency.

    2. `upsertBuilder` (mutation): Takes `walletAddress`, `currentStep`, `completedSteps`. If builder exists (by wallet index), patches currentStep and completedSteps. If not, inserts new record. Lowercases wallet address.

    3. `completeStep` (mutation): Takes `walletAddress` and `stepNumber`. Looks up builder by wallet. If found, adds stepNumber to completedSteps (if not already present), updates currentStep to max(completedSteps) + 1 (capped at 4). If not found, creates new builder record with that step completed. This is the primary mutation the wizard UI will call.

    Use exact patterns from RESEARCH.md -- Convex query with index lookup and mutation with upsert pattern.

    **Step 4: Push schema to Convex**
    Run: `npx convex dev --once` (or equivalent) to push the schema and generate types in `convex/_generated/`.

    If Convex auth is not set up yet (user hasn't logged in), this task should still CREATE the files. The schema push can happen when the user runs `npx convex dev` manually. Document this in a comment at the top of schema.ts.
