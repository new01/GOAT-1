  <action>
    **Step 1: Update useWizard hook to accept wallet address and use Convex**

    Modify `hooks/use-wizard.ts`:

    Change signature to `useWizard(walletAddress?: string): WizardState & { isLoading: boolean }`.

    Add Convex integration:
    ```typescript
    import { useQuery, useMutation } from "convex/react";
    import { api } from "../convex/_generated/api";
    ```

    **When walletAddress is provided:**
    - Use `useQuery(api.builders.getByWallet, { walletAddress: walletAddress.toLowerCase() })` to load existing progress
    - Use `useMutation(api.builders.completeStep)` for completing steps
    - Derive `currentStep` and `completedSteps` from the Convex query result
    - When query result is undefined (loading), return `isLoading: true`
    - When query result is null (no builder record), use defaults (currentStep: 0, completedSteps: [])
    - `completeCurrentStep()` calls the Convex mutation AND updates local optimistic state
    - `goToStep()` works locally (view navigation only, no Convex write for navigation)

    **When walletAddress is NOT provided (Phase 1 default):**
    - Fall back to local useState (same behavior as Plan 02)
    - This allows the wizard to function without wallet connection
    - `isLoading` is always false in this mode

    **Key behaviors:**
    - Convex `useQuery` provides real-time reactivity -- if another tab completes a step, this tab updates automatically
    - Optimistic local state prevents UI lag while mutation is in-flight
    - Wallet address is always lowercased before Convex operations

    **Step 2: Update wizard-shell to support wallet address**

    Modify `components/wizard/wizard-shell.tsx`:

    For Phase 1, the wallet address is not yet available (wallet connection is Phase 2). Add an optional `walletAddress` prop that defaults to undefined:

    ```typescript
    interface WizardShellProps {
      walletAddress?: string;
    }
    ```

    Pass `walletAddress` to the `useWizard` hook:
    ```typescript

    ```

    Show a loading skeleton or spinner when `wizard.isLoading` is true (brief flash while Convex query resolves).

    In Phase 2, the page.tsx will pass the connected wallet address to WizardShell. For now, it passes nothing (local state mode).

    **Important: Maintain backward compatibility**
    The wizard must still work WITHOUT a wallet address (local state mode). Phase 2 will wire the real wallet address. Do not break the Phase 1 placeholder behavior.
